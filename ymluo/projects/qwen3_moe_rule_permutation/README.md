# Qwen3 MoE Rule-Permutation Synthetic LM

This project tests whether MoE routing learns specialization when the next-token
rule itself depends on a high-level rule group.

Data design:

- there are `RULE_NUM_HIGH_UNITS` high-level slots;
- each slot is assigned to one of `RULE_NUM_GROUPS` rule groups;
- each rule is a random permutation over the content vocabulary;
- each block is generated by repeatedly applying the selected permutation:

```text
x[t + 1] = P_rule(x[t])
```

The task is still standard causal LM: the model predicts the actual next token
in the generated sequence. The metadata is used only for routing diagnostics or
optional oracle routing.

Run:

```bash
bash ymluo/projects/qwen3_moe_rule_permutation/scripts/run_train.sh
```

Background run:

```bash
bash ymluo/projects/qwen3_moe_rule_permutation/scripts/nohup_train.sh
```

Useful comparisons:

```bash
# Learned gate with gate inhibition.
RUN_NAME=learned-gate \
ROUTING_OVERRIDE=none \
GATE_INHIBITION_WEIGHT=0.05 \
bash ymluo/projects/qwen3_moe_rule_permutation/scripts/run_train.sh

# All tokens forced into one expert. If this works, the data is too easy.
RUN_NAME=all-to-one \
ROUTING_OVERRIDE=all_to_one \
GATE_INHIBITION_WEIGHT=0.0 \
bash ymluo/projects/qwen3_moe_rule_permutation/scripts/run_train.sh

# Oracle specialization upper bound: rule_group -> expert.
RUN_NAME=oracle-rule-group \
ROUTING_OVERRIDE=rule_group \
GATE_INHIBITION_WEIGHT=0.0 \
bash ymluo/projects/qwen3_moe_rule_permutation/scripts/run_train.sh
```

Capacity sweep:

```bash
MOE_INTERMEDIATE_SIZE=16 RUN_NAME=learned-i16 bash ymluo/projects/qwen3_moe_rule_permutation/scripts/run_train.sh
MOE_INTERMEDIATE_SIZE=32 RUN_NAME=learned-i32 bash ymluo/projects/qwen3_moe_rule_permutation/scripts/run_train.sh
MOE_INTERMEDIATE_SIZE=64 RUN_NAME=learned-i64 bash ymluo/projects/qwen3_moe_rule_permutation/scripts/run_train.sh
```

Important defaults:

```text
MOE_HEAD_LEVEL=false
MOE_NUM_EXPERTS_PER_TOK=1
MOE_USE_COMMON_EXPERT=false
MOE_INTERMEDIATE_SIZE=64
RULE_NUM_GROUPS=4
RULE_NUM_RULES_PER_GROUP=1
RULE_BLOCK_LEN=16
```

Main metrics:

- `same_rule_group`: same rule group -> same expert;
- `same_rule_id`: same concrete permutation rule -> same expert;
- `same_high`: same high-level slot -> same expert;
- `expert_load_by_layer`: per-layer expert load, avoiding misleading cross-layer averages;
- `oracle_*`: eval under forced `rule_group -> expert` routing;
- `all1_*`: eval under forced all-to-one routing.

Eval a checkpoint:

```bash
CKPT_FILE=ymluo/projects/qwen3_moe_rule_permutation/outputs/train/moe-rule-permutation/checkpoints/10000.pth \
bash ymluo/projects/qwen3_moe_rule_permutation/scripts/run_eval.sh
```
