# 面向端侧 MoE 部署的 Feature-Level Specialization：Feature 定义、结构设计与阶段实验结果

## 1. 核心结论：从现有瓶颈到本周方案

上一阶段分析已经明确指出，现有 flat MoE 在端侧 swapping 中面临两个核心瓶颈：第一，直接预测下一 token 的 expert 激活并不稳定，因为 router 使用含 residual 的完整 token 表征，expert id 容易被 token-level identity 主导，而不是稳定对应可预测的 feature；第二，即使 expert 预测达到 100% 准确，动态换入一个完整 expert 的参数量仍然过大，在当前硬件存算速度比下很难把传输时延完全隐藏在计算之后。这两个问题的共同根因是现有 MoE 没有形成足够明确的 **feature-level specialization**：expert 没有稳定对应可预测、可复用、可拆分的 feature 计算。

本阶段的方案动机是利用语言数据和模型内部计算中天然存在的 Zipf / long-tail 分布，推动 expert 按真正的 feature 发生 specialization。真实数据中的 feature 不是均匀出现的：少数高频 feature 在大量 token / context 中反复出现，大量低频 feature 只在特定上下文中出现。因此，我们可以让高频 feature 对应 common experts 并常驻 HBM，用稍高的常驻参数比例换取动态 swap 参数量的大幅下降；同时，通过真正的 feature-level 分发，让 expert routing 更接近可预测的 feature activation，而不是只依赖完整 token 表征做隐式分发。

本周文档给出三项进展：

```text
1. 定义 feature:
   用训练后参数矩阵的谱方向定义模型内部 feature，
   并用奇异值放大后的 activation strength 衡量 token 激活了哪些 feature。

2. 验证 feature 分布:
   在 Qwen3-0.6B 上观察到明确的 feature activation 长尾结构，
   证明模型内部计算本身存在 hot feature / tail feature 分层。

3. 推进模型结构:
   从上一版 H-MoE 的隐式层次结构，推进到显式 Frequency-Based MoE，
   并完成 H-MoE 大模型训练结果与 F-MoE 初步 loss 结果验证。
```

这条路线的目标非常明确：通过 feature-level specialization，让高频 feature 对应常驻 computation，长尾 feature 对应动态 load-on-demand computation，从而同时提升 expert routing 的可预测性，并降低每个 token 需要动态换入的 expert 参数量。

## 2. Feature 定义与分布性质

我们把训练后参数矩阵中的谱方向定义为模型内部 feature。考虑模型中的一个线性参数矩阵 $W\in\mathbb{R}^{d\times d}$，对其做奇异值分解：

$$
W = U\Sigma V^\top,
$$

其中：

$$
U=[u_1,\dots,u_d],\qquad
V=[v_1,\dots,v_d],\qquad
\Sigma=\operatorname{diag}(\sigma_1,\dots,\sigma_d).
$$

对一个 token 的 hidden state $x\in\mathbb{R}^d$，线性层计算可以写成：

$$
xW
=
xU\Sigma V^\top
=
\sum_{i=1}^{d} (x^\top u_i)\sigma_i v_i^\top.
$$

这个分解给出了一个直接的 feature 解释：

```text
1. token 表征 x 先投影到输入侧谱方向 u_i；
2. 每个投影值被对应奇异值 sigma_i 放大；
3. 放大后的系数再加权输出侧方向 v_i，形成线性层输出。
```

因此，$u_i$ 是参数矩阵 $W$ 的第 $i$ 个输入侧 feature direction，$\sigma_i$ 是模型在该 direction 上学习到的放大强度，$v_i$ 是该 feature 对输出空间的贡献方向。

给定 token 表征 $x$，定义其在第 $i$ 个 feature 上的 raw projection：

$$
p_i(x;W)=x^\top u_i.
$$

考虑奇异值放大后，定义 feature activation：

$$
a_i(x;W)=\sigma_i p_i(x;W)=\sigma_i x^\top u_i.
$$

为了统计“一个 token 激活了哪些 feature”，我们使用非负 energy：

$$
e_i(x;W)=a_i(x;W)^2
=
\sigma_i^2 (x^\top u_i)^2.
$$

这个定义同时包含两件事：

```text
token 是否查询到该 feature direction:      (x^T u_i)^2
模型是否在该 direction 上学习到强放大:      sigma_i^2
```

因此，feature activation 不是外部人工标签，而是模型真实前向计算中的内生结构。

### 2.1 Token 激活 Feature 的判定方式

一个 token 通常不会只激活单个 feature，而是激活一组 feature。我们使用 energy coverage threshold 将连续 activation 转成可统计的 active feature set。

给定阈值 $\tau\in(0,1)$，例如 $\tau=0.9$，对所有 feature energy 从大到小排序：

$$
e_{\pi_1}(x;W)\ge e_{\pi_2}(x;W)\ge\dots\ge e_{\pi_d}(x;W).
$$

定义最小整数 $m_\tau(x;W)$ 满足：

$$
\frac{\sum_{j=1}^{m_\tau(x;W)} e_{\pi_j}(x;W)}
{\sum_{i=1}^{d} e_i(x;W)}
\ge
\tau.
$$

于是 token $x$ 在参数矩阵 $W$ 上激活的 feature 集合定义为：

$$
\mathcal{A}_\tau(x;W)
=
\{\pi_1,\dots,\pi_{m_\tau(x;W)}\}.
$$

直观地说，当 $\tau=90\%$ 时，我们认为 token 激活了那些合计解释其 90% feature energy 的最小 feature 集合。这个定义保留了语言 feature 的组合性，同时给出了清晰的稀疏统计口径。

基于数据集 $\mathcal{D}=\{x_1,\dots,x_n\}$，每个 feature 的激活频率定义为：

$$
f_i(W;\tau)
=
\frac{1}{n}\sum_{t=1}^{n}
\mathbf{1}\{i\in\mathcal{A}_\tau(x_t;W)\}.
$$

$f_i$ 回答的是：在全部 token / context 中，有多少比例会激活第 $i$ 个 feature。将 $f_i$ 从大到小排序后，如果少数 feature 频繁激活、大量 feature 低频激活，就说明模型内部 feature usage 呈现长尾结构。

### 2.2 Qwen3-0.6B 上的 Feature 长尾激活证据

我们在 Qwen3-0.6B 上统计了 attention 中每一层 $q/k/v/o$ projection 矩阵的 feature activation。实验覆盖全 28 层、每层 4 个 projection，共 112 个参数矩阵，阈值取 $\tau=0.9$。

本组实验得到四条关键结论：

```text
1. Feature activation frequency 呈现明确长尾结构。
2. q/k 中存在接近全局共享的高频 feature。
3. Attention 中 token 的 feature activation 是 sparse 的。
4. Feature 使用频率和 feature activation strength 都具有结构性差异。
```

#### 2.2.1 Feature 激活频率呈现明确长尾

实验结果显示，feature activation frequency 呈现明确的长尾结构：少数 feature 在大量 token 中反复激活，大量 feature 只在少数 token 中激活。这个分布不是均匀分布，而是典型的 hot feature + long-tail feature 结构。

![Mean feature activation frequency, linear y](../fdong/experiments/svd_feature_activation_qwen3_0p6b_full/mean_feature_activation_frequency_linear.png)

图 1 使用 linear y-axis 展示平均 feature activation frequency。这个视角直接展示了系统设计关心的结构：左侧少数 feature 是高频 common feature，右侧大量 feature 是低频 tail feature。

为了观察尾部分布，我们也绘制了 log-log 图：

![Mean feature activation frequency, log-log](../fdong/experiments/svd_feature_activation_qwen3_0p6b_full/mean_feature_activation_frequency_loglog.png)

图 2 显示 $q/k$ 的 Zipf-like 形态最清晰；$v/o$ 同样长尾，但更平滑。这与 attention 中不同矩阵的功能一致：$q/k$ 更接近检索和匹配，因此更容易形成清晰的 feature query 分布；$v/o$ 更接近信息承载和输出混合，因此 feature activation 更分散。

#### 2.2.2 高频 Feature 在 q/k 中接近全局共享

按 28 层平均，最高频 feature 的激活频率如下：

| matrix | top-1 feature activation frequency | mean feature activation frequency |
|---|---:|---:|
| q_proj | 99.97% | 14.72% |
| k_proj | 99.91% | 12.90% |
| v_proj | 85.67% | 28.69% |
| o_proj | 96.57% | 31.09% |

这组数字说明，q/k 中存在几乎被所有 token 激活的 high-frequency feature，同时平均 feature activation frequency 明显低于 top-1。这正是长尾结构的典型表现：少数 feature 极热，大量 feature 明显更冷。

对端侧 MoE 来说，这个现象非常关键。高频 feature 对应 resident common computation；低频 feature 对应 dynamic unique expert。系统不需要把这两类计算混在同一个完整 expert 里一起搬运。

#### 2.2.3 每个 Token 的 Feature 激活是能量稀疏的

在 rank 1024 的 feature basis 上，每个 token 覆盖 90% feature energy 所需的平均 feature 数如下：

| matrix | average active features per token | active ratio |
|---|---:|---:|
| q_proj | 150.8 | 14.7% |
| k_proj | 132.1 | 12.9% |
| v_proj | 293.8 | 28.7% |
| o_proj | 318.4 | 31.1% |

![Average active feature count by layer](../fdong/experiments/svd_feature_activation_qwen3_0p6b_full/avg_active_feature_count_by_layer.png)

这个结果说明，token 的 feature activation 不是 one-hot，但也远不是 dense full-rank。尤其在 q/k 中，一个 token 平均只需要约 13% 到 15% 的 feature 即可覆盖 90% energy。这可以称为 **energy-level sparse activation**。

#### 2.2.4 激活强度同样具有结构差异

除了统计“多少 token 激活某个 feature”，我们还统计了“激活该 feature 的 token 在该 feature 上的平均 scaled activation strength”：

![Mean activation strength](../fdong/experiments/svd_feature_activation_qwen3_0p6b_full/mean_avg_activation_strength_linear.png)

这进一步说明，feature 之间不仅使用频率不同，激活强度也不同。后续可以用这两个维度区分不同类型的 feature：

```text
high-frequency high-strength feature:
  适合进入 common expert / resident computation。

low-frequency high-strength feature:
  通常对应特定上下文、特定能力或长尾模式，
  适合进入 unique expert / dynamic swapping。
```

## 3. 模型结构：从 H-MoE 到 Frequency-Based MoE

上一阶段我们基于数据频率分布的 intuition 直接设计了一个 straight-forward 结构：Hierarchical Common-Unique MoE。它把 expert 组织成 global common、group common、leaf unique 三层，动机是让高频 feature 对应 common path，让低频 feature 对应 leaf unique path。

这一版 H-MoE 已经具备正确的系统形态，但它仍然是隐式利用 feature 频率：每一级 router 都直接使用完整 token 表征 $h$ 做分发，并没有显式告诉 router 哪些方向是高频 feature、哪些方向是中频或低频 feature。它依赖模型训练过程自己学出这种分层。

第 2 章给出了新的条件：我们已经能够用参数矩阵的谱方向定义 feature，并统计每个 feature 的激活频率。因此，本周进一步引入 **Frequency-Based MoE**：直接用 feature 频率分段构造 router 输入，让高频 feature 不分发，中频 feature 粗粒度分发，低频 feature 细粒度分发。

### 3.1 上一版结构：Hierarchical Common-Unique MoE

上一版 H-MoE 可以快速概括为三层：

```text
global common expert:
  所有 token 都经过，承载最高频、全局共享 feature。

group common expert:
  token 被 top-level router 分到某个 group；
  该 group 内的 token 共享 group common expert，
  承载中频、组内共享 feature。

leaf unique expert:
  token 在 group 内继续被 leaf router 分发；
  leaf expert 承载低频、长尾、上下文特定 feature。
```

关键点是：**所有 router 和 experts 的输入都是同一个 post-attention residual 表征 $h$。** common expert 的输出不会再作为 group router 的输入；group common 的输出也不会再作为 leaf router 的输入。router 是并行地从同一个 $h$ 中读信息，只是结构上分成了 group 和 leaf 两级。

下面用 $3\times 3$ 的例子说明上一版结构：3 个 top-level groups，每个 group 内 3 个 leaf unique experts。图中只展开第 2 个 group。

![3 by 3 Hierarchical Common-Unique MoE](assets/hier_moe_3x3.svg)

形式化地，设 $h\in\mathbb{R}^{d}$ 是 Transformer block 中经过 attention residual 后、进入 MoE/FFN 的 token 表征。上一版 H-MoE 包含一个 global common expert $C_0$，$G$ 个 group common experts $\{C_g\}_{g=1}^{G}$，以及每个 group 内 $M$ 个 leaf unique experts $\{U_{g,m}\}_{m=1}^{M}$。

group router 和 leaf router 都以完整表征 $h$ 为输入：

$$
g^\*(h)=\arg\max_g \operatorname{softmax}(W_g h)_g,
$$

$$
m^\*(h)=\arg\max_m \operatorname{softmax}(W_{u,g^\*}h)_m.
$$

top-1 routing 下，该层输出为：

$$
\operatorname{HMoE}(h)
=
C_0(h)
+
C_{g^\*(h)}(h)
+
U_{g^\*(h),m^\*(h)}(h).
$$

部署策略为：

$$
\mathcal{R}_{\mathrm{resident}}
=
\{C_0\}\cup\{C_g\}_{g=1}^{G},
\qquad
\mathcal{R}_{\mathrm{dynamic}}(h)
=
\{U_{g^\*(h),m^\*(h)}\}.
$$

这个结构已经把 common path 和 leaf unique path 拆开，是“竖着看”的层次化结构：global common、group common、leaf unique 从结构上对应不同频率的 feature。

### 3.2 本次新增结构：Frequency-Based MoE

Frequency-Based MoE 把上一版 H-MoE “横着拉平”来看：不再只依赖 router 从完整表征中隐式学习频率层次，而是直接用第 2 章定义的 feature activation frequency 对 feature basis 排序，再按频率段构造不同粒度的 router。

以某个参数矩阵 $W_O$ 为例，可以使用其输入侧 feature basis 对 token 表征 $h$ 做投影。设 $W_O$ 的 feature 维度为 $r$，按全数据 feature activation frequency 从高到低排序后得到：

$$
\phi_1,\phi_2,\dots,\phi_r.
$$

对当前 token，计算 scaled feature activation：

$$
z_i(h)=\sigma_i h^\top \phi_i.
$$

然后按频率 rank 将 feature 拆成三段：

$$
B_{\mathrm{high}}=\{1,\dots,\lfloor 0.1r\rfloor\},
$$

$$
B_{\mathrm{mid}}=\{\lfloor 0.1r\rfloor+1,\dots,\lfloor 0.2r\rfloor\},
$$

$$
B_{\mathrm{low}}=\{\lfloor 0.2r\rfloor+1,\dots,r\}.
$$

直观上，如果 $W_O$ 有 100 个 feature directions，那么：

```text
0%-10% 高频 feature:
  不做分发，所有 token 都进入 common expert。

10%-20% 中频 feature:
  用这 10 维 feature activation 做一个 10 x n1 的 gate，
  route 到少量 mid-frequency experts。

20%-100% 低频 feature:
  用这 80 维 feature activation 做一个 80 x n2 的 gate，
  route 到更细粒度的 low-frequency experts。
```

这里的实现原则是：**feature projection 只用于产生 gate，expert 的输入始终是原来的 $h$。** 也就是说，进入 common expert、mid experts、low experts 的表征，都是同一个 post-attention residual 表征 $h$；这与普通 MoE / 上一版 H-MoE 的 expert 输入保持一致。

![Frequency-Based MoE](assets/frequency_based_moe.svg)

形式化地，定义三段 feature activation 向量：

$$
z_{\mathrm{mid}}(h)=z_{B_{\mathrm{mid}}}(h)\in\mathbb{R}^{0.1r},
\qquad
z_{\mathrm{low}}(h)=z_{B_{\mathrm{low}}}(h)\in\mathbb{R}^{0.8r}.
$$

中频 gate 为：

$$
a(h)=\operatorname{softmax}(z_{\mathrm{mid}}(h)^\top G_{\mathrm{mid}}),
\qquad
G_{\mathrm{mid}}\in\mathbb{R}^{0.1r\times n_1}.
$$

低频 gate 为：

$$
b(h)=\operatorname{softmax}(z_{\mathrm{low}}(h)^\top G_{\mathrm{low}}),
\qquad
G_{\mathrm{low}}\in\mathbb{R}^{0.8r\times n_2}.
$$

top-1 routing 下：

$$
i^\*(h)=\arg\max_i a_i(h),
\qquad
j^\*(h)=\arg\max_j b_j(h).
$$

设 common expert 为 $C$，中频 experts 为 $\{M_i\}_{i=1}^{n_1}$，低频 experts 为 $\{L_j\}_{j=1}^{n_2}$。该层输出为：

$$
\operatorname{FreqMoE}(h)
=
C(h)
+
M_{i^\*(h)}(h)
+
L_{j^\*(h)}(h).
$$

如果使用 gate weight，则写为：

$$
\operatorname{FreqMoE}(h)
=
C(h)
+
a_{i^\*}(h)M_{i^\*(h)}(h)
+
b_{j^\*}(h)L_{j^\*(h)}(h).
$$

对应的驻留策略为：

```text
common expert:
  最高频 feature 对应的共享计算，常驻 HBM。

mid-frequency experts:
  中频 feature 对应的粗粒度分发，可以常驻或作为小规模预取对象。

low-frequency experts:
  低频长尾 feature 对应的细粒度分发，是主要 dynamic prefetch / swap 对象。
```

expert 尺寸也随 feature 频率递减：

$$
\operatorname{size}(C)
>
\operatorname{size}(M_i)
>
\operatorname{size}(L_j).
$$

这使得动态换入的 leaf / low-frequency expert 更小，更容易把传输隐藏在 common path 和 mid-frequency path 的计算之后。

### 3.3 两个结构的关系

上一版 H-MoE 和本次 Frequency-Based MoE 不是同一个实现。它们的关系是同一系统目标下的两种视角：

```text
上一版 H-MoE:
  竖着看。先设计 global common -> group common -> leaf unique 的层次结构；
  每个 router 都用完整 token 表征 h；
  依赖模型自己把高频 / 中频 / 低频 feature 分到对应层级。

本次 Frequency-Based MoE:
  横着看。先用 SVD feature activation frequency 显式拆分 feature space；
  高频 feature 不分发，中频 feature 粗分发，低频 feature 细分发；
  router 输入是对应 frequency band 的 feature activation，expert 输入仍是 h。
```

因此，Frequency-Based MoE 可以看成对上一版 H-MoE 的显式化扩展：上一版结构已经给出了 common / group / leaf 的系统形态；本次结构把“为什么这样分层”进一步落实到可计算的 feature frequency band 上。

```text
H-MoE 的 global common
  对应 Frequency-Based MoE 的 0%-10% 高频 feature，不分发，常驻。

H-MoE 的 group common
  对应 Frequency-Based MoE 的 10%-20% 中频 feature，粗粒度分发。

H-MoE 的 leaf unique
  对应 Frequency-Based MoE 的 20%-100% 低频 feature，细粒度分发并动态换入。
```

换言之，上一版 H-MoE 是结构先验，本次 Frequency-Based MoE 是 feature 定义之后的显式路由实现。二者共同服务同一个端侧目标：高频计算常驻，长尾计算按需换入，把动态搬运量从完整 expert 降到更小的低频 expert。

下面两章围绕这一目标给出阶段验证。第 4 章验证 H-MoE 在 loss 和下游性能不下降的前提下，能否把动态 swapping 对象从完整 expert 拆到更小的 leaf expert，并观察当前结构下 leaf expert 的切换频率。第 5 章验证 Frequency-Based MoE 的显式 feature-frequency routing 是否可以接入训练并保持 loss 接近 baseline。后续还将继续补充低频 expert routing 的可预测性测试，这是决定 prefetch 策略收益的关键指标。

## 4. 20B Token H-MoE 实验结果

在上一版 Hierarchical Common-Unique MoE 结构上，我们进一步完成了基于 Qwen backbone 的 12-layer / 8-groups / 4-leaf-experts 训练与评测。该模型在 20B token 训练后表现正常，能够同时保持语言建模能力、下游任务表现和健康的 expert routing。

### 4.1 训练 Loss 正常下降

训练 loss 随 seen tokens 稳定下降，到 20B token 附近约为 4.0。曲线整体平滑、趋势清晰，说明 H-MoE 结构在 20B token 训练规模下是稳定可训练的。

### 4.2 下游任务表现提升

相比上一版 `moe_0522` 中的小规模 H-MoE 结果，本次 20B token H-MoE 在可比任务上有明确提升；新增任务也一并列入表中：

| task | metric | `moe_0522` H-MoE | 20B H-MoE | change |
|---|---|---:|---:|---:|
| ARC-Easy | accuracy | 32.66% | 36.66% | +4.00 pp |
| HellaSwag | accuracy | 25.92% | 26.82% | +0.90 pp |
| LAMBADA OpenAI | accuracy | - | 23.64% | - |
| LAMBADA OpenAI | perplexity | - | 213.02 | - |

这些结果说明，H-MoE 的 common-unique 结构保持了模型基本能力；随着训练 token 数增加，模型能力按预期提升。

### 4.3 Expert Routing 健康

本次结构包含 8 个 top-level groups，每个 group 内 4 个 leaf experts，共 32 个 leaf experts。routing 统计显示，绝大多数层中 group 和 leaf experts 都被稳定使用：

```text
Layer 0-9:
  group active: 8/8
  leaf active: 32/32

Layer 10:
  group active: 8/8
  leaf active: 31/32
```

从 entropy 和 top expert 占比看，主要层中的 experts 均被充分使用。前 11 层平均统计为：

| metric | value |
|---|---:|
| mean group normalized entropy | 0.906 |
| mean leaf normalized entropy | 0.893 |
| mean group top-1 usage | 27.16% |
| mean leaf top-1 usage | 11.98% |

这说明 H-MoE 在真实数据上形成了健康的 routing 结构：experts 被充分使用，同时不同 expert 之间存在一定流量差异，为后续 resident common path 与 dynamic leaf path 的系统拆分提供了模型侧基础。

### 4.4 Expert Swapping 指标

在端侧部署中，group common experts 会常驻 HBM，因此 group switch rate 不是主要瓶颈。真正需要关注的是 leaf unique experts 的动态换入。

当前朴素 H-MoE 结构已经把动态换入对象从完整 expert 降到更小粒度的 leaf expert，具备小粒度 leaf swapping 的结构基础。逐 token 的 leaf switch rate 在浅层约为 0.94，在深层约为 0.53，全层平均约为 0.82；leaf mean run length 从浅层约 1.06 增加到深层约 1.87。

这组数字说明，H-MoE 已经完成了参数粒度拆分：真正动态换入的是更小的 leaf expert，而不是完整 expert。同时，朴素 H-MoE 的 router 仍然使用完整 token 表征 $h$，没有显式利用 feature frequency band，因此对 leaf expert 可预测性和专家交换频率的改善较小。第 3.2 节提出的 Frequency-Based MoE 正是针对这一点：用显式 feature-band activation 产生 gate，让低频 leaf expert routing 更贴近可预测的 feature activation。

## 5. F-MoE 初步实验结果

基于第 3.2 节的 Frequency-Based MoE 设计，我们进一步比较了新的 F-MoE 结构与 baseline MoE 的训练 loss。本次 F-MoE 采用本文定义的 **SV route + full input** 结构。

该设置含义如下：

```text
SV route:
  用于分发的每个 frequency band 不是单纯 feature projection，
  而是乘过对应奇异值后的 feature activation strength。

full input:
  feature-band projection 只用于产生 gate；
  真正进入 expert 的仍然是完整 post-attention residual 表征 h。
```

这正好对应本文的核心设计原则：router 可以看显式 feature activation，但 expert 本身仍然处理完整表征，避免把进入 expert 的信息过早截断。

由于 baseline 训练时间更长，图中只展示 baseline 与 F-MoE 都有结果的相同 token 区间，确保比较发生在相同训练进度上。

![F-MoE SV route full input vs baseline loss](frequency-moe-results/frequency_moe_sv_full_vs_baseline_loss.png)

在共同的 3.046B token 区间内，两条 loss 曲线基本贴近：

| model | tokens | final loss in shared span | loss at 3.0B tokens |
|---|---:|---:|---:|
| baseline 4e top1 | 3.046B | 3.5086 | 3.5035 |
| F-MoE: SV route + full input | 3.046B | 3.5072 | 3.5183 |

这个结果说明，显式 feature-frequency routing 接入 MoE 训练后，loss 与 baseline 保持接近。它把第 2 章的 feature 定义真正用于 router 输入，同时保持 expert 输入为完整表征，是从 H-MoE 结构先验迈向显式 feature-specialized MoE 的第一步。

### 5.1 下游任务与 Routing 可预测性

F-MoE 的下一步目标不是单纯降低 loss，而是在模型能力保持稳定的前提下，让低频 expert routing 更适合端侧 prefetch。当前新增评测显示，新结构在下游任务上与 baseline 保持接近：

| model | downstream average |
|---|---:|
| baseline | 0.4088 |
| F-MoE: SV route + full input | 0.4052 |

这说明显式 feature-frequency routing 接入后，整体任务能力保持在同一水平。

在 routing 可预测性上，我们分别看两个指标：

| prediction setting | baseline | F-MoE: SV route + full input |
|---|---:|---:|
| same token, predict routing across layers | 0.767 | 0.58-0.61 |
| same layer, predict next-token routing | 0.483 | 0.491-0.498 |

这组结果需要按端侧 prefetch 的目标来理解。Same-token 跨层预测下降，说明 F-MoE 的不同 group 不再简单复制 baseline 中由完整 token 表征主导的跨层 routing 相关性；而 same-layer next-token routing 从 0.483 提升到 0.491-0.498，说明与实际 prefetch 直接相关的下一 token routing 可预测性已经出现小幅提升。

因此，F-MoE 当前达到了阶段目标：在 loss 和下游任务保持接近的前提下，把 router 输入从完整 token 表征推进到显式 feature activation，并开始改善 next-token expert routing 的可预测性。后续需要继续扩大训练规模，并进一步分析不同 frequency group 的 routing activation 与 prefetch 命中率。

## 6. 总结与下一步

本周工作的意义在于把端侧 MoE swapping 从“预测下一个 expert”推进到“定义、测量并利用模型内部 feature 频率分布”。我们给出了可计算的 feature 定义，验证了真实模型中 feature activation 的长尾结构，并把这一结构落实到两级模型方案中：H-MoE 给出 common / group / leaf 的层次化结构先验，Frequency-Based MoE 则进一步用显式 feature frequency band 做 routing，使高频 feature 常驻、低频 feature 动态换入这一系统目标可以直接进入模型设计。

从实验结果看，20B token 的 H-MoE 训练稳定、下游任务表现按预期提升、expert routing 健康，说明 common-unique 结构可以在真实模型上正常训练并保持模型能力；F-MoE 的 loss 与下游任务表现接近 baseline，并且在 same-layer next-token routing 预测上从 0.483 提升到 0.491-0.498，说明显式 feature-frequency routing 可以接入训练流程，并开始改善与 prefetch 直接相关的可预测性。下一步工作将继续测试 Frequency-Based MoE 的 routing activation 变化、不同 frequency group 的 prefetch 命中率、expert swapping 频率和下游任务性能，并进一步扩展到更大模型和更长训练规模。
