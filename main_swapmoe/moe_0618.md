# 0618 会议材料草稿：基于 Feature 频率分解的 Frequency-Based MoE

## 1. 本阶段目标：从 Expert Swapping 转向 Feature-Level Specialization

上一阶段已经明确，端侧 MoE 部署的核心瓶颈不是单纯“能否预测下一个 expert”，而是现有 MoE 的 expert 组织方式没有形成足够清晰的 **feature-level specialization**。

具体来说，flat MoE 在端侧 load-on-demand 部署中同时遇到两个问题：

```text
1. 可预测性问题:
   router 使用完整 token 表征做分发，expert id 容易被 token identity 和 residual 表征主导；
   因此跨 token / 跨层预测 expert activation 时，预测目标并不稳定。

2. 搬运粒度问题:
   一个 flat expert 同时包含 common feature 和 unique feature 的计算；
   即使 expert 预测完全准确，每次仍需要搬运一个完整 expert，参数量过大。
```

因此，本阶段的研究重点从“预测 flat expert id”推进到“让 expert 对应可解释、可拆分、可管理的 feature 计算”。我们的目标是让高频 feature 对应常驻计算，让中低频 feature 对应更细粒度的动态 expert，从而在保持模型能力的前提下，降低端侧逐 token decode 时需要动态换入的参数量。

0615 文档已经给出了 feature 定义与 Frequency-Based MoE 的初步结构。本次 0618 文档继续沿着这条线推进，重点讨论三件事：

```text
1. 如何基于 feature 频率设计 Frequency-Based MoE；
2. 如何理解模型内部表征空间中的 feature；
3. 如何评价该结构的模型能力、routing 可预测性和 swapping 参数量收益。
```

## 2. 模型结构：Frequency-Based MoE

本阶段采用显式的 **Frequency-Based MoE**。它的核心思想是：用 feature activation frequency 决定 routing 粒度。

设进入 FFN/MoE 的 post-attention residual 表征为 $h\in\mathbb{R}^{d}$。对当前层选定的 projection 矩阵，例如 $W_O$，得到输入侧 feature basis $\{\phi_i\}_{i=1}^{r}$ 及奇异值 $\{\sigma_i\}_{i=1}^{r}$。当前 token 的 feature activation 为：

$$
z_i(h)=\sigma_i h^\top \phi_i.
$$

根据全数据统计得到的 feature activation frequency，对 feature directions 从高频到低频排序，并划分为三段：

$$
B_{\mathrm{high}}=\{0\%-10\%\},
\qquad
B_{\mathrm{mid}}=\{10\%-20\%\},
\qquad
B_{\mathrm{low}}=\{20\%-100\%\}.
$$

三段 feature 分别承担不同系统角色：

```text
High-frequency feature:
  所有 token 都高概率需要，直接进入 common expert；
  common expert 常驻 HBM，不参与动态 swapping。

Mid-frequency feature:
  使用中频 feature activation 做粗粒度 gate；
  对应少量 mid-frequency experts，可根据系统预算选择常驻或半常驻。

Low-frequency feature:
  使用低频 feature activation 做细粒度 gate；
  对应大量更小的 low-frequency experts，是主要 load-on-demand / prefetch 对象。
```

需要强调的是，feature projection 只用于 routing。真正进入 expert 的输入始终是完整表征 $h$，而不是被切分后的局部 feature component：

$$
\operatorname{FMoE}(h)
=
C(h)
+
M_{i^\*(h)}(h)
+
L_{j^\*(h)}(h).
$$

其中：

$$
i^\*(h)=\arg\max_i \operatorname{Gate}_{\mathrm{mid}}(z_{B_{\mathrm{mid}}}(h))_i,
$$

$$
j^\*(h)=\arg\max_j \operatorname{Gate}_{\mathrm{low}}(z_{B_{\mathrm{low}}}(h))_j.
$$

这一结构保留了普通 MoE 的 full hidden-state expert computation，同时让 router 显式基于 feature frequency 做分发。它的系统含义是：高频 feature 的计算常驻，低频 feature 的计算被拆成更小粒度的 dynamic expert。

## 3. 表征空间理解：Feature 是参数矩阵中的谱方向

我们把训练后参数矩阵中的谱方向看作模型内部 feature。以某一层 attention output projection 矩阵 $W_O$ 为例，对其做奇异值分解：

$$
W_O = U\Sigma V^\top.
$$

对输入表征 $h$，线性层计算可以写成：

$$
hW_O
=
hU\Sigma V^\top
=
\sum_i (h^\top u_i)\sigma_i v_i^\top.
$$

这个分解提供了一个直接解释：

```text
u_i:
  输入侧 feature direction。

h^T u_i:
  当前 token 在该 feature direction 上的投影。

sigma_i:
  模型训练后赋予该 feature direction 的放大强度。

sigma_i h^T u_i:
  当前 token 在该 feature 上的 activation strength。

v_i:
  该 feature 对输出表征空间的贡献方向。
```

因此，feature 不是外部人工标签，而是模型参数和 token 表征共同决定的内部计算结构。一个 token 激活哪些 feature，可以用奇异值加权后的 energy 来定义：

$$
e_i(h;W_O)=\sigma_i^2(h^\top u_i)^2.
$$

将所有 feature 的 energy 从大到小排序后，取能够覆盖总能量 $\tau$ 的最小集合，例如 $\tau=90\%$，即可得到当前 token 激活的 feature set：

$$
\mathcal{A}_\tau(h;W_O)
=
\left\{
\pi_1,\dots,\pi_m
\right\},
\qquad
\frac{\sum_{j=1}^{m}e_{\pi_j}(h;W_O)}
{\sum_i e_i(h;W_O)}
\ge \tau.
$$

这一口径的意义在于：它把“feature”从抽象概念变成了可计算对象。我们可以进一步统计每个 feature 在真实 token 上被激活的频率，并据此区分 high-frequency feature、mid-frequency feature 和 low-frequency feature。

## 4. 关键观察：不同层的 Feature Basis 并不天然对齐

Frequency-Based MoE 使用每层参数矩阵的谱方向来构造 routing feature。这个设计让 router 显式看到每层真实的 feature activation，也带来一个重要观察：不同层的 $W_O$ 有不同的 SVD basis，因此同名 frequency band 或同名 group 并不天然表示同一个语义坐标。

我们对 Qwen3-0.6B 各层 attention output projection 的奇异子空间做了分析。结果显示，不同层之间的输入侧 feature basis 差异很大。以 layer 4 和 layer 10 为例，右奇异向量 $V$ 的前 5 个方向几乎不对齐：

| right singular vector | layer4-layer10 abs cosine |
|---|---:|
| sv#1 | 0.029 |
| sv#2 | 0.009 |
| sv#3 | 0.020 |
| sv#4 | 0.013 |
| sv#5 | 0.005 |

进一步看子空间对齐，top 10% input-side subspace 的 layer4-layer10 adjusted overlap 仅约 $0.0295$，10%-20% subspace 的 adjusted overlap 仅约 $0.0168$。这说明跨层同名 feature band 基本不是同一个坐标系。

![right V top5 alignment](o_proj_subspace_alignment/right_V_input_side_top5_vector_alignment.png)

这一结果说明 F-MoE 确实在按照每层自己的 feature basis 做 specialization，而不是把不同层都压到同一套 token-level shortcut 上。换言之，第 4 层的第 $k$ 个 group 与第 10 层的第 $k$ 个 group 不必表示同一个 feature 子空间；这正是模型按不同层功能形成 specialization 的表现。

这也给出下一步 predictor 的设计原则：

```text
训练时:
  保持每层自己的 feature basis，让模型自由学习表达能力。

预测 / prefetch 时:
  不直接假设跨层同名 group 对齐；
  将前层表征投影到目标层的 feature basis 上，
  再预测目标层的 local group / expert id。
```

具体地，当我们用第 4 层预测第 10 层的 expert activation 时，可以使用第 10 层 $W_O$ 分解出的目标层 feature basis。设第 10 层某个 frequency band 的 feature directions 为 $\Phi_{10,B}$，奇异值为 $\Sigma_{10,B}$，第 4 层表征为 $h_4$，则 prediction feature 可以写成：

$$
\tilde z_{4\to 10,B}
=
h_4\Phi_{10,B}\Sigma_{10,B}.
$$

也就是说，预测器不再问“第 4 层第 $k$ 个 group 能否预测第 10 层第 $k$ 个 group”，而是把第 4 层表征放到第 10 层的 feature 坐标系里，直接读取与第 10 层 routing 决策相关的 feature activation。这样做既保留了训练时逐层 specialization 的自由度，又为推理时跨层 prefetch 提供了对齐后的预测输入。

## 5. 阶段实验结果

### 5.1 Loss 与下游任务性能

当前 F-MoE 采用 **SV route + full input** 设置：

```text
SV route:
  router 输入使用乘过奇异值的 feature activation strength，
  而不是未加权的 raw projection。

full input:
  expert 输入仍然使用完整 post-attention residual 表征 h，
  不把 expert 输入截断成局部 feature component。
```

在共同训练区间内，F-MoE 与 baseline loss 保持接近：

| model | tokens | final loss in shared span | loss at 3.0B tokens |
|---|---:|---:|---:|
| baseline 4e top1 | 3.046B | 3.5086 | 3.5035 |
| F-MoE: SV route + full input | 3.046B | 3.5072 | 3.5183 |

![F-MoE SV route full input vs baseline loss](frequency-moe-results/frequency_moe_sv_full_vs_baseline_loss.png)

下游任务平均分也保持在同一水平：

| model | downstream average |
|---|---:|
| baseline | 0.4088 |
| F-MoE: SV route + full input | 0.4052 |

这说明显式 feature-frequency routing 可以接入训练流程，并且没有破坏模型的基本语言建模能力和下游任务能力。

### 5.2 Expert Activation 可预测性

端侧 swapping 真正关心的是：低频 dynamic expert 是否能够被提前预测并 prefetch。F-MoE 的可预测性评估需要匹配它的结构特点：由于不同层使用各自的 feature basis 做分发，跨层预测不应直接比较同名 group，而应把前层表征投影到目标层 routing feature basis 后，再预测目标层 local expert id。

当前已有的局部 group-to-group 结果可以作为这一判断的直接证据：

| prediction setting | baseline | F-MoE: SV route + full input |
|---|---:|---:|
| same token, local group-to-group cross-layer prediction | 0.767 | 0.58-0.61 |
| same layer, next-token routing prediction | 0.483 | 0.491-0.498 |

这个结果与第 4 节的结构观察一致：F-MoE 的目标不是让不同层的同名 group 共享同一个 token-level shortcut，而是让每层根据自己的 feature basis 做 specialization。因此，局部同名 group-to-group 并不是最终 prefetch predictor 应采用的输入形式；跨层预测时需要使用目标层 feature basis 做对齐后的 prediction feature。

因此，0618 阶段的结论是：

```text
F-MoE 没有破坏 expert activation 的可预测性。

不同层 feature basis 不共享坐标系，说明模型形成了逐层 feature specialization；
预测器应使用目标层 feature basis 读取前层表征中的相关信息。

下一步 prefetch predictor 应预测目标层 local expert id，
输入应是投影到目标层 routing feature basis 后的 activation。
```

### 5.3 Swapping 参数量建模

F-MoE 的系统收益来自把动态换入对象从完整 flat expert 拆成 low-frequency expert。

设 attention 主干单层参数近似为 $4d^2$。flat MoE 有 $N$ 个 experts，每个 expert 为 $d\to kd\to d$，则单层总参数为：

$$
P_{\mathrm{flat}}
\approx
4d^2+2Nkd^2.
$$

端侧部署时，若只常驻 attention、router 和每层一个高频 expert，则 flat MoE 的 resident 参数为：

$$
R_{\mathrm{flat}}
\approx
4d^2+2kd^2,
$$

逐 token 需要动态换入的 expert 参数量为：

$$
S_{\mathrm{flat}}
\approx
2kd^2.
$$

对于 F-MoE，设 common expert expansion 为 $k_c$，mid-frequency experts 数量为 $n_m$、expansion 为 $k_m$，low-frequency experts 数量为 $n_l$、expansion 为 $k_l$。则单层总参数近似为：

$$
P_{\mathrm{fmoe}}
\approx
4d^2
+2k_cd^2
+2n_mk_md^2
+2n_lk_ld^2.
$$

若端侧常驻 attention、common expert 和 mid-frequency experts，仅对 low-frequency experts 做 load-on-demand，则常驻参数比例为：

$$
\rho_{\mathrm{fmoe}}
=
\frac{
4+2k_c+2n_mk_m
}{
4+2k_c+2n_mk_m+2n_lk_l
}.
$$

逐 token 动态换入参数量为：

$$
S_{\mathrm{fmoe}}
\approx
2k_ld^2.
$$

相对 flat MoE 的逐 token swapping 参数量比例为：

$$
\eta_{\mathrm{swap}}
=
\frac{S_{\mathrm{fmoe}}}{S_{\mathrm{flat}}}
=
\frac{k_l}{k}.
$$

因此，只要 low-frequency expert 的 expansion $k_l$ 显著小于 flat expert 的 expansion $k$，F-MoE 就可以在保持高频 / 中频计算常驻的同时，把每个 token 需要动态换入的参数量压到原来的很小一部分。

在与上一阶段相同的系统目标下，如果 low-frequency expert 设计为 flat expert 的 $1/8$ 大小，则逐 token 动态 expert 参数搬运量可以降低到 baseline 的约 $12.5\%$。这意味着 swapping 不再是“搬运完整 expert”的带宽瓶颈，而是可以尝试在 resident common / mid-frequency computation 期间预取更小的 low-frequency expert。

## 6. 结论

本阶段的核心进展是把 MoE swapping 的问题从 expert id 预测推进到了 feature-level specialization。

第一，我们用参数矩阵的 SVD 方向给出了可计算的 feature 定义，并把 token 在 feature 上的 activation strength 定义为奇异值加权后的投影。这个定义使得 high-frequency / low-frequency feature 可以被实际统计和用于模型设计。

第二，我们用 Frequency-Based MoE 将 feature 频率直接接入 router：高频 feature 不分发并对应常驻 common computation，中频 feature 做粗粒度分发，低频 feature 做细粒度分发并作为主要 dynamic swapping 对象。

第三，当前实验已经证明 F-MoE 的训练 loss 和下游任务表现与 baseline 保持接近；不同层 feature basis 不对齐进一步说明模型确实形成了逐层 feature specialization。下一步跨层 prefetch predictor 将使用目标层 $W_O$ 的 feature basis 对前层表征做投影，从而直接预测目标层 local expert activation。

第四，在系统建模上，F-MoE 把每个 token 的动态搬运对象从完整 flat expert 降到更小的 low-frequency expert。只要 low-frequency expert 的 expansion 明显小于 flat expert，就可以显著降低逐 token swapping 参数量，为端侧 decode 中将传输隐藏在 resident computation 后面创造条件。

下一步工作聚焦两点：

```text
1. 完成 F-MoE checkpoint 的实际参数量口径统计，
   给出 resident ratio 与 dynamic swap ratio 的最终数字。

2. 完成基于目标层 feature basis 投影的 prediction-time routing predictor 评测，
   将预测准确率进一步转化为 prefetch hit rate 和端侧 latency 模型收益。
```
